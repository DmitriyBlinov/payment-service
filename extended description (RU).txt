На технологиях JDK 18, Jetty 11, Gradle 7 и MongoDB 5 реализовать сервер приёма информации о платежах:

- servlet принимает post-запросы с информацией о платежах, в теле каждого ожидается информация об одном платеже: json-сообщение с полями id (строка), userId (число) и sum (число).
- сервер синхронно отвечает на каждый такой запрос json-сообщением с единственным полем status (число): 0, если это новый платёж (информации о платеже с таким id ранее не поступала) или 1, если такой платёж уже был.
- сервер сохраняет информацию о каждом новом платеже в отдельном документе в хранилище; при получении запроса о платеже, который уже был - новой записи не делает.
- требуется корректно обрабатывать ситуацию одновременного получения запросов с одинаковым новым id от разных клиентов
- для взаимодействия с хранилищем требуется использовать MongoDB Java Driver.

Нужно снабдить проект инструкцией по запуску сервера и тестом для отправки запросов.
